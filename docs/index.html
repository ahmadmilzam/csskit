What is CSSKit?

CSSKit is a powerful, SASS-Based, BEM, OOCSS framework designed with scalability and performance in mind. It has a tiny footprint (The Essential Kit comes in at ~1KB, gzipped), and can be scaled as much or as little as you need.

OOCSS is really about a way to writing CSS that avoid duplicating code by separating styles into letered abstractions.

CSSKit provides a solid architectural foundation upon which you can build any size or style of website or app. CSSKit is what you make of it, it helps you get started, and then gets out of your way.

CSSKit is a framework in its truest sense. It is not a UI Toolkit, with opinionated views on design; it is not one giant download, with hundreds of unnecessary lines of CSS; it is not a 1001-UI-components-plus-the-kitchen-sink, with loads of bloated rules. CSSKit's focus is less upon the actual code within it, but upon the principles of a solid, scalable CSS architecture.

OFF by default!

In the interests of reducing the amount of code in your project, all of CSSKit's module-variants are turned off by default. This gives CSSKit two layers of reduced bloat:

  1. You only include the specific files you need, so youâ€™re immediately starting with a very trimmed down codebase.
  2. Any variants are switched off by default, so if you just want buttons, you don't get every different size and permutation of them unless you explicitly tell the framework you want them.

To turn features on, just set their switches to true before you @import the file: 


  $enable-btn--full:    true;
  $enable-btn--large:   true;
  @import "4-objects/objects.buttons";

Say goodbye to bloated old UI Toolkits and opinionated libraries.
