What is CSSKit?

CSSKit is a powerful, Sass-based, BEM, OOCSS framework designed with scalability and performance in mind. It has a tiny footprint (the Starter Kit comes in at 1KB, gzipped), and can be scaled as much or as little as you need.

CSSKit provides a solid architectural foundation upon which you can build any size or style of website or app. CSSKit is what you make of it-it helps you get started, and then gets out of your way.

CSSKit is a framework in its truest sense. It is not a UI Toolkit, with opinionated views on design; it is not one giant download, with hundreds of unnecessary lines of CSS; it is not a 1001-UI-components-plus-the-kitchen-sink, with loads of bloated rules. It is a collection of tiny, composable, configurable, interdependent modules that you can piece together how you want to, when you want to. CSSKit's focus is less upon the actual code within it, but upon the principles of a solid, scalable CSS architecture.

OFF by default!

In the interests of reducing the amount of code in your project, all of CSSKit's module-variants are turned off by default. This gives CSSKit two layers of reduced bloat:

  1. You only include the specific files you need, so youâ€™re immediately starting with a very trimmed down codebase.
  2. Any variants are switched off by default, so if you just want buttons, you don't get every different size and permutation of them unless you explicitly tell inuitcss you want them.

To turn features on, just set their switches to true before you @import the file: 

          
            $enable-btn--full:    true;
            $enable-btn--large:   true;
            @import "bower_components/objects/objects.buttons";

Say goodbye to bloated old UI Toolkits and opinionated, monolithic libraries.

Components Grid

In a world of agile development and super-tablet-multi-magic-laptop-phones, the best layouts can't be contained in a single framework or technique. CSS Libraries are a bloated mess of opinions about how to do your job. Why let the table-saw tell you where to put the kitchen? If we can create them ourself?

The only component that CSSKit has is Grids Component. But not a bunch of html classes, only collection of mixin to help you create your own grid.

2 columns layout example

        
          .layout__aside{
            @media #{$large-up}{
              @include grid-column(4);
            }
          }

          .layout__main{
            @media #{$large-up}{
              @include grid-column(8, true);
            }
          }

3 columns layout example

        
          .layout{
            @include grid-row();

            .layout{
              @include grid-row($container-width, nested);
            }
          }
            .layout__item{
            @include grid-column($columns:false, $position:true);

            .layout--3col > &{
              @media #{$large-up}{
                @include grid-column($columns:4);
              }
